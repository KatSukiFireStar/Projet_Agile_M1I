Class: Adaptateur

Adaptateur(option)

Classe Adaptateur qui permet d'adapter les données en fonction de nos besoins En entrée, on aura toutes les données 'en vrac', ensuite, la classe va valider les données et les sauvegarder dans le 'Session Storage' ainsi, dans la page 'jeux.html', on pourra récupérer les données et les utiliser comme on le souhaite.

Methods

adapterFichierJson()

Méthode qui adapte le fichier dans les deux cas. Si l'on lance une nouvelle partie, le fichier est simplement sauvegardé dans le 'session storage' sinon, dans le cas d'une reprise de partie, on va aussi sauvegarder le fichier (qui sera différent) mais on va aussi récupérer la position de la tache sur laquelle les joueurs s'étaient arrêtés.

  • main.js, line 25

adapterMode()

Méthode qui adapte le mode de jeux dans tous les cas. Si le mode sélectionné par les joueurs est invalide alors le mode par défaut est attribué (Strict). Sauvegarde la valeur dans le 'session storage'.
Source:
  • main.js, line 60

adapterNbJoueurs()

Méthode qui adapte le nombre de joueurs de la partie. Si le nombre est invalide alors, on sauvegarde 2 joueurs. Sauvegarde la valeur dans le 'session storage'.
Source:
  • main.js, line 73

adapterNomJoueurs()

Méthode qui adapte la liste avec les noms des joueurs de la partie. Si cette liste est vide, elle est invalide et alors, on attribue une liste avec 2 noms de joueurs. Sauvegarde la valeur dans le 'session storage'.
